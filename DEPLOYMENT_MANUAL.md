
# 단\\독주택 안전점검 배정 지도 운영 매뉴얼

이 문서는 `safety_single_house_app` 배포 시 유의사항과 월별 데이터 업데이트 절차를 정리한 관리자용 매뉴얼입니다.

## 1. 배포 시 유의사항

### 1.1 배포 방식

현재 앱은 로컬 PC에서 실행되는 웹 애플리케이션입니다.

사용자는 `안전점검배정.exe`를 실행하고, 브라우저에서 지도 화면을 사용합니다.

```text
http://127.0.0.1:8000/
```

사내망에서 여러 사용자가 같이 접속하려면 관리자 PC에서 앱을 실행한 뒤, 다른 사용자는 관리자 PC 주소로 접속합니다.

```text
http://관리자PC이름:8000/
http://관리자PC_IP:8000/
```

### 1.2 EXE 파일만 배포하면 안 되는 이유

현재 `안전점검배정.exe`는 실행 파일 내부에 모든 데이터를 포함하지 않습니다. 같은 폴더에 있는 HTML, JSON, API 키, 원본 엑셀, geocoding 캐시를 읽습니다.

따라서 사용자에게는 EXE 단독 파일이 아니라 `safety_single_house_app` 폴더 전체를 전달해야 합니다.

### 1.3 필수 배포 파일

배포 폴더에는 아래 파일과 폴더가 반드시 있어야 합니다.

```text
safety_single_house_app/
├── 안전점검배정.exe
├── app.py
├── open_map.bat
├── daejeon_map.html
├── admin.html
├── process_geocode.json
├── api_keys.json
├── requirements.txt
├── README.md
├── DEPLOYMENT_MANUAL.md
├── API_map/
└── geocodes/
```

각 파일 역할은 아래와 같습니다.

| 항목                     | 역할                                     |
| ------------------------ | ---------------------------------------- |
| `안전점검배정.exe`     | 사용자 실행 파일                         |
| `app.py`               | FastAPI 서버 및 데이터 처리 로직         |
| `open_map.bat`         | 관리자 PC에서 서버 실행용 배치 파일      |
| `daejeon_map.html`     | 지도 화면                                |
| `admin.html`           | 월별 데이터 갱신 관리자 화면             |
| `process_geocode.json` | 지도 표시 및 배정에 사용하는 가공 데이터 |
| `api_keys.json`        | 네이버 지도 및 Geocoding API 키          |
| `requirements.txt`     | Python/bat 실행 시 필요한 패키지 목록    |
| `API_map/`             | 월별 원본 안전점검 엑셀 보관 폴더        |
| `geocodes/`            | 주소 좌표 변환 캐시                      |

### 1.4 배포 제외 권장 파일

아래 파일은 운영에 필요하지 않거나 개발/테스트용이므로 사용자 배포 폴더에 포함하지 않습니다.

```text
__pycache__/
build/
dist/
*.spec
*_test_*.txt
*_report.json
fix_*.py
check_*.py
```

현재 개발/테스트/빌드 산출물은 운영 폴더 밖의 아래 경로에 보관합니다.

```text
C:\DA\safety_single_house_app_dev_archive
```

### 1.5 API 키 보안

`api_keys.json`에는 네이버 API 인증 정보가 들어 있습니다.

주의사항:

- 외부 공개 저장소에 업로드 금지
- 메신저, 이메일 등으로 무분별하게 공유 금지
- GitHub에 업로드할 경우 `.gitignore`에 반드시 추가
- 키가 유출되면 네이버 클라우드 플랫폼에서 즉시 재발급

### 1.6 네이버 지도 URL 등록

네이버 지도 SDK는 등록된 Web 서비스 URL에서만 정상 로드됩니다.

로컬 실행용 URL:

```text
http://127.0.0.1:8000
http://localhost:8000
```

사내망 공유용 URL:

```text
http://관리자PC이름:8000
http://관리자PC_IP:8000
```

사용자 PC 또는 관리자 PC 주소가 바뀌면 네이버 클라우드 Maps Application의 Web 서비스 URL도 추가 등록해야 합니다.

### 1.7 방화벽 및 포트

기본 포트는 `8000`입니다.

다른 PC에서 접속하지 못하는 경우 아래 항목을 확인합니다.

1. 관리자 PC에서 앱이 실행 중인지 확인
2. 접속 주소가 맞는지 확인
3. Windows 방화벽에서 8000 포트가 허용되어 있는지 확인
4. 같은 사내망에 연결되어 있는지 확인
5. 네이버 지도 Web 서비스 URL에 해당 주소가 등록되어 있는지 확인

## 2. 실행 방법

### 2.1 사용자 단독 실행

사용자가 자기 PC에서만 사용할 경우:

1. `safety_single_house_app` 폴더 전체를 PC에 복사
2. `안전점검배정.exe` 실행
3. 브라우저가 자동으로 열리면 지도 사용
4. 사용 종료 시 EXE 창 닫기

### 2.2 관리자 PC에서 사내 공유 실행

여러 사용자가 같은 앱에 접속해야 하는 경우:

1. 관리자 PC에 `safety_single_house_app` 폴더 배치
2. `open_map.bat` 실행
3. 콘솔 창에 표시되는 접속 URL 확인
4. 사용자에게 아래 주소 중 하나 공유

```text
http://관리자PC이름:8000/
http://관리자PC_IP:8000/
```

관리자 PC가 꺼지거나 서버 창을 닫으면 사용자는 접속할 수 없습니다.

## 3. 월별 데이터 업데이트 방법

### 3.1 원본 엑셀 파일 준비

월별 원본 엑셀은 아래 파일명 규칙을 지켜야 합니다.

```text
H[센터코드]_YYYY.MM_안전점검.xlsx
```

예시:

```text
H051_2026.08_안전점검.xlsx
H071_2026.08_안전점검.xlsx
H072_2026.08_안전점검.xlsx
H073_2026.08_안전점검.xlsx
H074_2026.08_안전점검.xlsx
H075_2026.08_안전점검.xlsx
```

파일명 형식이 맞지 않으면 앱에서 월별 데이터로 인식하지 못합니다.

### 3.2 원본 엑셀 위치

새 월 엑셀은 반드시 아래 폴더에 넣습니다.

```text
safety_single_house_app\API_map
```

기존 월 파일을 삭제할 필요는 없습니다. 여러 월 파일이 같이 있어도 월 드롭다운에서 선택할 수 있습니다.

### 3.3 관리자 화면에서 갱신

1. `안전점검배정.exe` 또는 `open_map.bat` 실행
2. 관리자 페이지 접속

```text
http://127.0.0.1:8000/admin
```

3. `API_map 기준 갱신 실행` 클릭
4. 처리 완료 메시지 확인
5. 지도 화면 새로고침
6. 월 드롭다운에서 추가한 월 선택

### 3.4 관리자 화면에서 직접 업로드

엑셀 파일을 `API_map`에 직접 복사하지 않고 관리자 화면에서 업로드할 수도 있습니다.

1. `http://127.0.0.1:8000/admin` 접속
2. 월별 엑셀 파일 선택
3. `업로드 및 갱신` 클릭
4. 처리 완료 메시지 확인
5. 지도 화면 새로고침

업로드된 파일은 자동으로 `API_map/` 폴더에 저장됩니다.

### 3.5 데이터 갱신 시 내부 처리 흐름

관리자가 갱신을 실행하면 앱은 다음 순서로 처리합니다.

```text
API_map 폴더의 H*.xlsx 검색
    ↓
파일명 규칙 확인
    ↓
안전점검 시트 읽기
    ↓
용도 = 단독주택 데이터만 필터링
    ↓
센터/월/주소/오더 수 집계
    ↓
기존 geocodes 캐시 확인
    ↓
신규 주소는 네이버 Geocoding API 호출
    ↓
process_geocode.json 재생성
    ↓
지도 화면에서 새 월 데이터 사용 가능
```

### 3.6 업데이트 후 확인 사항

월별 데이터 갱신 후 아래 항목을 확인합니다.

- 새 월이 월 드롭다운에 표시되는지
- 센터별 주소 수와 오더 수가 원본 엑셀과 크게 차이 없는지
- 지도가 정상 표시되는지
- 대전/계룡/충남 외 이상 좌표가 과도하게 발생하지 않는지
- 인원 입력 후 배정 실행이 정상 동작하는지
- 엑셀 저장이 정상 동작하는지

## 4. 문제 발생 시 점검 순서

### 4.1 지도가 안 뜨는 경우

1. 인터넷 연결 확인
2. 네이버 지도 API 키 확인
3. 네이버 클라우드 Web 서비스 URL 등록 확인
4. 브라우저 새로고침
5. `api_keys.json` 존재 여부 확인
6. 실행 URL이 `http://127.0.0.1:8000` 또는 등록된 URL인지 확인

### 4.2 새 월 데이터가 안 보이는 경우

1. 파일명이 `H[센터코드]_YYYY.MM_안전점검.xlsx` 형식인지 확인
2. 파일이 `API_map/`에 있는지 확인
3. 관리자 화면에서 `API_map 기준 갱신 실행`을 눌렀는지 확인
4. 처리 완료 후 지도 화면을 새로고침했는지 확인
5. 원본 엑셀에 `안전점검` 시트가 있는지 확인
6. `용도` 열에 단독주택 데이터가 있는지 확인

### 4.3 Geocoding 결과가 이상한 경우

1. 도로명 주소와 지번 주소가 원본에 모두 있는지 확인
2. 주소가 대전/계룡/충남 권역인지 확인
3. 같은 지번이 타지역에도 존재하는지 확인
4. 필요 시 도로명 주소 기준으로 재조회
5. `geocodes/` 캐시와 `process_geocode.json` 재생성

### 4.4 다른 PC에서 접속이 안 되는 경우

1. 관리자 PC에서 서버가 실행 중인지 확인
2. `open_map.bat` 창이 닫히지 않았는지 확인
3. 관리자 PC IP가 바뀌지 않았는지 확인
4. Windows 방화벽에서 8000 포트 허용 여부 확인
5. 같은 사내망에 연결되어 있는지 확인

## 5. 운영 원칙

- 월별 원본 엑셀은 `API_map/`에 누적 보관
- 데이터 갱신은 관리자 화면에서 수행
- 배포 전 `api_keys.json` 및 원본 엑셀 보안 확인
- 공개 GitHub에는 내부 데이터와 API 키 업로드 금지
- 사용자는 폴더 전체를 받아 실행
- EXE만 단독 배포하지 않음
