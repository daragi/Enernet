# 단독주택 안전점검 배정 지도

서비스처리센터별 안전점검 엑셀 데이터를 지도에 시각화하고, 단독주택 오더를 인원별로 배정하기 위한 로컬 웹 애플리케이션입니다.

## 프로젝트 개요

본 프로젝트는 월별 안전점검 엑셀 파일에서 `안전점검` 시트의 단독주택 데이터를 추출하고, 네이버 지도와 Geocoding API를 활용해 주소를 지도 위에 표시합니다. 이후 서비스처리센터별 총 오더 수와 입력 인원 수를 기준으로 인당 목표 오더를 계산하고, 주소 간 인접성과 지번 흐름을 고려해 담당 구역을 배정합니다.

주요 목표는 다음과 같습니다.

- 단독주택 안전점검 대상 주소를 지도 기반으로 확인
- 서비스처리센터와 월별 데이터 필터링
- 인원 수 입력에 따른 오더 배정 자동화
- 배정 결과를 지도 하이라이트와 주소 목록으로 확인
- 배정 결과를 엑셀 파일로 저장
- 관리자 화면을 통한 월별 데이터 갱신

## 문제 정의

기존 안전점검 배정은 주소와 오더 수를 엑셀에서 직접 확인해야 했기 때문에 다음 문제가 있었습니다.

- 같은 지번 또는 인접 지번의 오더를 시각적으로 파악하기 어려움
- 센터별 전체 오더 수 대비 인원별 균등 배정 기준을 수동 계산해야 함
- 지리적으로 멀리 떨어진 섬 형태의 오더가 특정 인원에게 비효율적으로 배정될 수 있음
- 월별 데이터가 추가될 때마다 지도와 배정 데이터 갱신 절차가 필요함

이 앱은 위 문제를 해결하기 위해 원본 엑셀, 네이버 Geocoding, 지도 시각화, 배정 로직을 하나의 화면에서 연결합니다.

## 주요 기능

- 월별 엑셀 데이터 로드
- `용도` 값이 단독주택인 데이터 필터링
- 지번 주소 및 도로명 주소 기반 Geocoding
- 네이버 지도 위 점 표시
- 지적편집도 표시
- 서비스처리센터별 필터링
- 인원별 오더 배정
- 인원 클릭 시 해당 담당 주소만 강조
- 우측 패널에서 주소 상세 확인
- 주소 클릭 시 지도 이동
- 배정 결과 `.xlsx` 저장
- 관리자 페이지에서 월별 엑셀 업로드 및 데이터 재생성

## 실행 방법

### 사용자 실행

사용자는 배포 폴더 전체를 받은 뒤 `안전점검배정.exe`를 실행합니다.

```text
안전점검배정.exe
```

실행 후 브라우저가 자동으로 열리며, 기본 주소는 아래와 같습니다.

```text
http://127.0.0.1:8000/
```

주의: 현재 EXE는 같은 폴더의 `daejeon_map.html`, `process_geocode.json`, `api_keys.json`을 읽습니다. 따라서 EXE 파일 하나만 복사하면 정상 동작하지 않습니다. 배포 시 폴더 전체를 전달해야 합니다.

### 관리자 실행

관리자는 `open_map.bat` 또는 `안전점검배정.exe`를 실행합니다.

```text
open_map.bat
```

관리자 PC에서 접속:

```text
http://127.0.0.1:8000/
```

관리자 페이지:

```text
http://127.0.0.1:8000/admin
```

사내망 공유 주소:

```text
http://관리자PC이름:8000/
http://관리자PC_IP:8000/
```

다른 사용자가 접속하려면 관리자 PC가 켜져 있고 서버가 실행 중이어야 합니다.

## 월별 데이터 갱신 방법

월별 원본 엑셀은 `API_map/` 폴더에 넣습니다.

파일명 규칙:

```text
H[센터코드]_YYYY.MM_안전점검.xlsx
```

예시:

```text
H051_2026.08_안전점검.xlsx
H071_2026.08_안전점검.xlsx
H072_2026.08_안전점검.xlsx
H073_2026.08_안전점검.xlsx
H074_2026.08_안전점검.xlsx
H075_2026.08_안전점검.xlsx
```

관리자 갱신 절차:

1. 새 월 엑셀 파일을 `API_map/`에 복사
2. `안전점검배정.exe` 또는 `open_map.bat` 실행
3. `http://127.0.0.1:8000/admin` 접속
4. `API_map 기준 갱신 실행` 클릭
5. 갱신 완료 후 지도 화면 새로고침
6. 월 드롭다운에서 새 월 선택

관리자 화면에서 엑셀 파일을 직접 선택해 `업로드 및 갱신`을 실행할 수도 있습니다. 이 경우 업로드된 파일은 `API_map/`에 저장되고 `process_geocode.json`이 재생성됩니다.

## 인원 배정 기준

인원 배정의 기본 전제는 다음과 같습니다.

```text
서비스처리센터별 총 오더 수 / 입력 인원 수 = 인당 목표 오더 수
```

배정 로직은 다음 기준을 함께 고려합니다.

- 같은 지번 또는 인접 지번의 주소를 최대한 연속적으로 배정
- 전체 오더 수가 인원별로 크게 벌어지지 않도록 균등성 유지
- 가까운 주소 묶음을 우선 배정
- 떨어진 섬 오더는 가장 가까운 담당자 후보에게 배정
- 배정 결과는 지도 점 색상과 동선으로 확인

현재 로직은 완전한 차량 경로 최적화가 아니라, 안전점검 구역 배정에 필요한 균등성과 지리적 인접성을 동시에 맞추는 휴리스틱 방식입니다.

## 배포 주의사항

- `API_KEY` 저장소 업로드 금지
- 원본 엑셀에 개인정보 또는 내부 업무 데이터가 포함된 경우 업로드 금지
- `process_geocode.json`에도 주소 및 좌표 정보가 포함될 수 있으므로 공개 저장소 업로드 전 검토 필요
- 네이버 클라우드 Maps Application에 실행 URL 등록 필요
- 로컬 실행 URL: `http://127.0.0.1:8000`, `http://localhost:8000`
- 사내망 공유 URL: `http://관리자PC이름:8000`, `http://관리자PC_IP:8000`
- 8000 포트가 방화벽에 막혀 있으면 다른 PC에서 접속 불가

## 실행 검증

최종 배포 전 아래 항목을 확인합니다.

- `안전점검배정.exe` 실행 시 브라우저 자동 오픈
- `/` 지도 화면 정상 로드
- `/admin` 관리자 화면 정상 로드
- `/safety_assignment_app_data.json` 데이터 응답 정상
- 센터 및 월 드롭다운 정상 표시
- 배정 실행 정상 동작
- 엑셀 저장 정상 동작
